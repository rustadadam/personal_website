# Code Review Checklist

## Code Quality & Interview Readiness
- [ ] Code is readable, well-structured, and strictly typed (TypeScript).
- [ ] No duplicated logic. Abstract repeated code into utility functions.
- [ ] State management is clear and easy to explain (e.g., using standard React hooks over complex external stores unless necessary).
- [ ] Tailwind classes are used correctly without bloated inline styles.

## Document Upkeep
- [ ] Update `@database_schema.md` if data models changed.
- [ ] Ensure all file names and folder structures match standard Next.js conventions.
